# 2D 平台游戏碰撞系统实现方案

基于开源平台游戏源码分析整理的三种主流碰撞架构。

---

## 一、架构 1：纯手动 AABB（无物理引擎）

**代表项目**：[super-mario-mimic](https://github.com/iam-veeramalla/super-mario-mimic)、[platform-game](https://github.com/dijith-481/platform-game)

### 核心思路

所有碰撞都是数学计算，没有物理引擎参与。每帧手动移动实体、检测矩形重叠、修正位置。

### 碰撞检测

```typescript
checkCollision(a, b): boolean {
    return a.x < b.x + b.width &&
           a.x + a.width > b.x &&
           a.y < b.y + b.height &&
           a.y + a.height > b.y;
}
```

### 平台碰撞方向判定

**优先级：先纵向（落地/撞头）后横向（墙壁）**

```typescript
handlePlatformCollisions(platforms: Platform[]): void {
    this.isGrounded = false;  // 每帧重置
    for (const platform of platforms) {
        if (!this.checkCollision(this, platform)) continue;

        if (this.velocityY > 0 && this.y < platform.y) {
            // 1. 顶部碰撞（落在平台上）
            this.y = platform.y - this.height;
            this.velocityY = 0;
            this.isGrounded = true;
        }
        else if (this.velocityY < 0 && this.y > platform.y) {
            // 2. 底部碰撞（头撞平台）
            this.y = platform.y + platform.height;
            this.velocityY = 0;
        }
        else if (this.velocityX > 0 && this.x < platform.x) {
            // 3. 右侧面碰撞
            this.x = platform.x - this.width;
            this.velocityX = 0;
        }
        else if (this.velocityX < 0 && this.x > platform.x) {
            // 4. 左侧面碰撞
            this.x = platform.x + platform.width;
            this.velocityX = 0;
        }
    }
}
```

### 踩踏判定

```typescript
// 碰到敌人时
if (this.collisionDetector.checkCollision(this.player, enemy)) {
    if (this.player.isJumping && this.player.velocityY > 0) {
        // 正在下落 + 跳跃状态 = 从上方踩到
        enemy.die();
        this.score += 100;
        this.player.bounce();  // velocityY = -200（小弹跳）
    } else {
        // 侧面/下方碰撞 = 玩家受伤
        this.playerHit();
    }
}
```

**踩踏判定条件**：`isJumping === true` 且 `velocityY > 0`（正在下落）

### 敌人巡逻

```typescript
// 敌人碰到墙壁或平台边缘时反向
if (this.x <= 0 || this.x + this.width >= worldWidth) {
    this.velocityX *= -1;
}

// 碰到平台侧面也反向
if (hitPlatformSide) {
    this.velocityX *= -1;
}
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 完全可控，行为可预测 | 需要自己实现所有物理（重力、摩擦等） |
| 无外部依赖 | 代码量大，复杂场景容易出 bug |
| 跨引擎通用 | 处理高速穿透（tunneling）较困难 |

## 二、架构 2：实体-瓦片分离检测

**代表项目**：[platfuse](https://github.com/praghus/platfuse)

### 核心思路

地图碰撞和实体碰撞是两套独立系统，瓦片碰撞用射线检测避免穿墙。

### 双系统分离

```
地图碰撞：collideWithTile()     → 瓦片级别精度，处理墙壁/地面
实体碰撞：collideWithEntity()   → 矩形碰撞，处理玩家/敌人/道具
```

### 瓦片射线检测

```typescript

```

### 实体碰撞

```typescript
collideWithEntity(entityA, entityB): boolean {
    // 标准 AABB 检测
    // 但只检测同一层级的实体
    // 通过 collision layer 过滤

```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 瓦片精度高，不卡墙角 | 射线检测开销比简单 AABB 大 |
| 不会高速穿透 | 实现复杂度中等 |
| 地图和实体解耦 | 需要维护两套碰撞逻辑 |

## 三、架构 3：物理引擎驱动（本项目）

**适用于**：Cocos Creator / Unity / Godot 等引擎

### 核心思路

利用引擎的 2D 物理系统（Box2D / PhysX），碰撞检测交给引擎，只写游戏逻辑。

### 碰撞回调

```typescript
onBeginContact(self: Collider2D, other: Collider2D, contact: IPhysics2DContact | null) {
    if (!contact) return;

    // 通过 contact.getWorldManifold() 获取碰撞法线
    const manifold = contact.getWorldManifold(null);
    const normal = manifold?.normal;

    // 通过 collider 属性分发逻辑
    if (other.sensor) {
        // 触发区域（梯子/伤害区）
    } else {
        // 物理碰撞体（地面/墙壁/敌人本体）
    }
}
```

### Sensor 分离碰撞类型

| 碰撞体类型 | sensor | 用途 |
|-----------|--------|------|
| 地面/墙壁 | `false` | 物理碰撞，引擎自动处理响应 |
| 梯子区域 | `true` | 触发梯子逻辑，无物理阻挡 |
| 敌人伤害区 | `true` | 触发受伤/踩踏判定 |
| 敌人本体 | `false` | 物理推开（可选） |

### 碰撞分组（位掩码）

```typescript
// 分组定义
const GROUP = {
    DEFAULT: 1 << 0,  // 1
    HERO:    1 << 1,  // 2
    WALL:    1 << 2,  // 4
    LADDER:  1 << 3,  // 8
    ENEMY:   1 << 4,  // 16
};

// 碰撞矩阵
// HERO  与 WALL + LADDER + ENEMY 碰撞
// ENEMY 与 WALL + HERO 碰撞
// WALL  与 HERO + ENEMY 碰撞
```

### 踩踏判定（利用碰撞法线）

```typescript
onBeginContact(_self: Collider2D, other: Collider2D, contact: IPhysics2DContact | null) {
    if (!contact) return;

    if (other.sensor) {
        this.onLadderEnter();
        return;
    }

    // 判断是否是敌人
    if (other.group === (1 << 4)) {  // ENEMY 组
        const manifold = contact.getWorldManifold(null);
        const normalY = manifold?.normal.y ?? 0;

        // 正在下落 + 碰撞法线朝上 = 踩踏
        if (this.rigidBody!.linearVelocity.y <= 0 && normalY > 0.5) {
            this.stompEnemy(other.node);
        } else {
            this.playerHit();
        }
        return;
    }

    // 地面/墙壁
    this.groundContactCount++;
    if (this.canReGround) this.isGrounded = true;
}
```

### 接地检测（计数法）

```typescript
// 多平台边缘情况：用计数而非布尔值
private groundContactCount: number = 0;

onBeginContact() {
    this.groundContactCount++;
    if (this.canReGround) this.isGrounded = true;
}

onEndContact() {
    this.groundContactCount--;
    if (this.groundContactCount <= 0) {
        this.groundContactCount = 0;
        this.isGrounded = false;
    }
}
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 碰撞响应自动处理 | 物理引擎行为不完全可控 |
| 开发效率高 | 需要理解分组和碰撞矩阵 |
| 天然支持复杂场景 | 调试物理问题比手动碰撞困难 |
| 射线检测 API 内置 | 高速物体可能穿透（需设置 CCD） |

## 四、关键设计对比

| 设计点 | 手动 AABB | 瓦片射线 | 物理引擎 |
|--------|-----------|----------|---------|
| 碰撞检测 | 矩形重叠判断 | 射线 + 矩形 | 引擎自动 |
| 碰撞响应 | 手动修正位置 | 手动修正 + 射线 | 引擎自动 |
| 接地检测 | 每帧重置布尔值 | 向下射线 | 计数法 |
| 踩踏判定 | `velocityY > 0` | 类似 | `linearVelocity.y + 碰撞法线` |
| 敌人巡逻 | 碰壁反向 | 碰壁反向 | 碰壁反向 |
| 方向判定 | 速度方向 + 位置关系 | 射线方向 | 碰撞法线 |
| 高速穿透 | 容易发生 | 射线避免 | 需开启 CCD |
| 开发成本 | 高（全部手写） | 中 | 低（逻辑为主） |

## 五、边缘卡顿（Edge Catching）

物理引擎驱动方案下的经典问题：两个地面碰撞体存在微小高度差时，玩家水平移动会被卡住。

### 问题示意

```
  玩家 →
  ┌──────┐
  │      │
  └──┬───┘
     │ 碰撞体A (y=0.05)
─────┘  │ 碰撞体B (y=0)
─────────┘
```

玩家经过接缝时，物理引擎检测到碰撞体A的**侧面**接触，将玩家向左推开 → 卡住。即使 0.05px 的差距也足以触发。

### 解决方案

**方案 A：碰撞体合并（从根源消除接缝）**

`TiledMapCollider` 的行扫描 + 纵向合并算法，将相邻同高瓦片合并为单个碰撞体，消除绝大部分接缝。

```
合并前（4个碰撞体，有接缝）   合并后（1个碰撞体，无缝）
┌──┬──┬──┬──┐               ┌──────────────┐
│  │  │  │  │               │              │
└──┴──┴──┴──┘               └──────────────┘
```

**方案 B：玩家底部使用圆形碰撞体**

用圆形碰撞体代替方形底部，圆弧面可滑过微小台阶：

```typescript
// 在玩家节点底部添加圆形碰撞体
const bottomCircle = node.addComponent(CircleCollider2D);
bottomCircle.radius = colliderHeight * 0.4;
bottomCircle.offset = new Vec2(0, -colliderHeight * 0.3);
bottomCircle.group = this.collisionGroup;
bottomCircle.apply();
```

**方案 C：单向平台（过滤侧面碰撞）**

只响应从上方落下的碰撞，忽略侧面接触：

```typescript
onBeginContact(_self: Collider2D, other: Collider2D, contact: IPhysics2DContact | null) {
    if (!contact) return;
    if (other.sensor) return;

    // 只响应顶部碰撞（法线朝上），忽略侧面
    const manifold = contact.getWorldManifold(null);
    const normalY = manifold?.normal.y ?? 0;
    if (normalY < 0.5) return;

    this.groundContactCount++;
    if (this.canReGround) this.isGrounded = true;
}
```

**方案 D：接地时对齐 Y 坐标**

玩家接地且纵向速度接近零时，将 Y 坐标对齐到合理精度：

```typescript
if (this.isGrounded && Math.abs(this.rigidBody.linearVelocity.y) < 1) {
    const pos = this.node.position;
    this.node.setPosition(pos.x, Math.round(pos.y * 20) / 20, pos.z);
}
```

### 推荐组合

| 优先级 | 方案 | 作用 |
|--------|------|------|
| 1 | 碰撞体合并（方案 A） | 消除大部分接缝 |
| 2 | 底部圆形碰撞体（方案 B） | 处理剩余微小台阶 |
| 3 | 单向平台（方案 C） | 可选，适合纯平台游戏 |
