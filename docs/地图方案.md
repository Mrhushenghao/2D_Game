# TiledMap 碰撞体生成方案

基于 TiledMap 编辑器的四种主流碰撞体生成方式，以及本项目的实现选择。

---

## 一、方案对比

| 方案 | 碰撞形状 | 编辑效率 | 引擎通用性 | 本项目实现 |
|------|---------|---------|-----------|-----------|
| Tile Layer 标记法 | 仅矩形 | 高 | 中 | `TiledMapCollider` |
| Object Layer 手绘形状 | 矩形 + 多边形 + 折线 | 中 | 最高 | `TiledMapObjectCollider` |
| Tileset Collision Editor | 矩形 + 多边形 | 前期高后期低 | 低 | 需自行解析 |
| 混合方案 | 全部 | 高 | 高 | 方案 1 + 方案 2 |

---

## 二、方案 1：Tile Layer 标记法

**脚本**：`TiledMapCollider.ts`

### 原理

在 TiledMap 中创建一个专门用于碰撞的 Tile Layer，用任意瓦片铺满需要碰撞的区域。代码扫描该图层中有瓦片的格子，通过行扫描 + 纵向合并算法生成最少数量的 `BoxCollider2D`。

### TiledMap 编辑操作

1. 创建新 Tile Layer，命名为 `main`（或自定义名称）
2. 在需要碰撞的区域铺设瓦片（瓦片样式无关，只判断有无）
3. 可与渲染图层分开，碰撞图层在游戏中不可见

### 合并算法

```
行扫描：逐行将连续的实心瓦片合并为水平条带
纵向合并：检查下方行是否有同宽条带，有则合并为更大矩形
```

### 生成结果

```
原始瓦片（4x2 格）        合并后（1 个碰撞体）
■■■■                     ┌──────────────┐
■■■■                     │              │
                          └──────────────┘
```

### 适用场景

- 平地、墙壁、矩形平台
- 大面积规则地形
- 碰撞体数量敏感的场景（合并后数量少）

### 局限

- 只能生成矩形碰撞体
- 无法处理斜坡、弧形等不规则地形

## 三、方案 2：Object Layer 手绘形状

**脚本**：`TiledMapObjectCollider.ts`

### 原理

在 TiledMap 的 Object Layer 中手动绘制矩形、多边形、折线等形状，代码读取后生成对应的 `BoxCollider2D` 或 `PolygonCollider2D`。

### TiledMap 编辑操作

1. 创建 Object Layer，命名为 `collision`（或自定义名称）
2. 用以下工具绘制碰撞形状：
   - **矩形工具**（R）：画矩形区域 → `BoxCollider2D`
   - **多边形工具**（P）：画多边形 → `PolygonCollider2D`
   - **折线工具**（L）：画折线 → 转为薄多边形碰撞体

### 支持的形状

| TiledMap 形状 | Cocos 碰撞体 | 说明 |
|--------------|-------------|------|
| 矩形 | `BoxCollider2D` | 平台、墙壁、触发区域 |
| 多边形 | `PolygonCollider2D` | 斜坡、弧形地面、不规则地形 |
| 折线 | `PolygonCollider2D`（薄多边形） | 薄碰撞线，厚度 2px |
| 椭圆 | `BoxCollider2D` | 按外接矩形处理 |

### 坐标转换

引擎 `getObjects()` 返回的坐标已转换为 Cocos 坐标系（Y 向上），无需手动翻转。

### 适用场景

- 斜坡、弧形地面
- 不规则地形轮廓
- 需要精确控制碰撞边界的场景

### 局限

- 手动绘制工作量大
- 碰撞形状与视觉瓦片需要手动保持同步

## 四、方案 3：Tileset Collision Editor

### 原理

在 Tileset 编辑器中为每种瓦片定义碰撞形状。放置瓦片时碰撞形状自动跟随。

### TiledMap 编辑操作

1. Tileset 面板 → 右键 → Edit Tileset
2. 选中瓦片 → 底部 Collision Editor
3. 为瓦片绘制碰撞形状（矩形/多边形）
4. View → Show Tile Collision Shapes 可在地图上预览

### 优势

- 碰撞形状跟随瓦片，画一次自动复用
- 碰撞与视觉天然同步
- 适合大量复用同种瓦片的场景

### 局限

- Cocos Creator 3.x **原生不解析** tileset 中的碰撞形状
- 需要手动解析 `.tmx` / `.tsx` 中每个 tile 的 `<objectgroup>` 数据
- Godot 4.x 原生支持此方案

### 未在本项目实现

Cocos Creator 不直接支持读取 tileset 碰撞数据，需要额外解析 XML，投入产出比较低。

## 五、方案 4：混合方案（本项目推荐）

### 组合策略

| 地形类型 | 使用方案 | 碰撞体 |
|---------|---------|--------|
| 平地、墙壁、平台 | Tile Layer 标记法 | `BoxCollider2D`（合并） |
| 斜坡、弧形、不规则地形 | Object Layer 多边形 | `PolygonCollider2D` |
| 触发区域（梯子、伤害区） | Object Layer 矩形 + Sensor | `BoxCollider2D`（sensor） |

### 使用方式

将 `TiledMapCollider` 和 `TiledMapObjectCollider` 挂载到同一个 TiledMap 节点上：

```
TiledMap 节点
├── TiledMap（组件）
├── TiledMapFix（组件）
├── TiledMapCollider（组件）         ← 读取 Tile Layer 生成矩形碰撞
└── TiledMapObjectCollider（组件）    ← 读取 Object Layer 生成多边形碰撞
```

### 注意事项

- 两个脚本的 `collisionGroupIndex` 保持一致（默认 2 = wall 组）
- Tile Layer 和 Object Layer 的碰撞区域不要重叠，避免重复碰撞
- Object Layer 中的梯子区域使用 Sensor 碰撞体，不需要这两个脚本处理

---

## 六、性能建议

- Tile Layer 方案通过合并算法已大幅减少碰撞体数量
- Object Layer 方案的碰撞体数量取决于手绘形状数量，通常较少
- 大地图建议分区域加载，不在视口内的碰撞体可通过 `allowSleep` 让物理引擎跳过计算
- 调试图形（`showDebug`）仅开发时开启，发布时关闭
